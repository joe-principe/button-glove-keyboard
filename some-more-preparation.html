Debouncing

This is the main issue I've been focusing on for the past couple of days. Whenever you press or release a button, the metal contacts of the button bounce on the metal leads. This causes the button to rapidly jump between the on and off states until the button is either fully on or fully off. This is an issue because these bounces can be picked up as undesired button presses/releases. We have to debounce the switch in order to get only the presses and releases we really want.

A button can either be debounced in software or hardware. With a software approach, we read the current state of the output, wait some time, then read the state again. If the states are the same, then most likely the change was a result of a button press and the output should be treated as correct. This effectively ignores any bounces by waiting until after they've concluded.

There are many different ways to debounc a switch with hardware. One of the ways I read about was to add an RC circuit in parallel with the switch and a Schmitt trigger in series with the resistor to act as the output. The RC portion should act as a low pass filter, effectively filtering out the bouncing of the switch. The Schmitt trigger helps give our output signal a sharper transition than it would have with just the resistor and capacitor.

I am currently debating how to approach this. My though is that if I can use both software and hardware, why not just do that? This could help ensure there is little to no bounce from key presses; however, it could also result in a lot of overhead. On the hardware side, adding more components to each switch would increase the cost and size of the keyboard glove significantly, which would make it a lot less portable than I'd like. There's probably a good middle ground that allows a little bounce on the hardware side that is then removed on the software side.

The Buttons Themselves

Pushbuttons are pretty simple, I've learned. They're basically just two wires with a metal contact that, when the button is pressed, connects the two wires together. There are four leads on the button to provide stability.

The buttons I'm planning on using are these ones from Gikfun that I found on Amazon. They seem fairly large, which is a plus because they'll be on the finger tips and their size should make pressing them down using the skin or other buttons a breeze.
